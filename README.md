## Supabase registration storage

This project now stores `/registration` submissions in Supabase Postgres through `POST /api/registrations`.

### Environment variables

Create `.env.local` (or copy from `.env.example`) and set:

```bash
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
```

### Database table

Create this table and strict security setup in Supabase SQL Editor:

```sql
create table if not exists public.registration_submissions (
  id bigint generated by default as identity primary key,
  full_name text not null,
  email text not null,
  skin_type text,
  preferred_category text,
  concerns text[] not null default '{}',
  selected_product_ids text[] not null default '{}',
  submitted_at timestamptz not null default now()
);

alter table public.registration_submissions enable row level security;

drop policy if exists "Allow anon insert registration submissions"
on public.registration_submissions;

drop policy if exists "Allow public insert registration submissions"
on public.registration_submissions;

revoke all on table public.registration_submissions from anon, authenticated;
grant insert, select on table public.registration_submissions to service_role;

create index if not exists idx_registration_submissions_email
  on public.registration_submissions(email);
```

Important:
- `SUPABASE_SERVICE_ROLE_KEY` must be stored only on the server (never expose it in `NEXT_PUBLIC_` env vars).
- Registration inserts happen through the server route (`/api/registrations`) using the service role key.
